第一章 引论

本课程讨论的内容

写出一个可以工作的程序并不够。如果这个程序在巨大的数据集上进行，那么运行时间就变成了重要的问题。

数学知识复习

  • 指数

\[ X^AX^B=X^{A+B}\\ \frac{X^A}{X^B}=X^{A-B}\\ (X^A)^B=X^{AB}\\ 2^N+2^N=2^{N+1} \]

  • 对数

在计算机科学中，除非有特殊的说明，所有对数都是以2为底的。

定义：\(X^A=B\)，当且仅当\(\log_XB=A\)

由此得到几个方便的等式： \[ \log_AB = \frac{\log_CB}{\log_CA}\ (C>0)\\ \log AB=\log A+\log B\\ \log\frac{A}{B}=\log A-\log B\\ \log(A^B)=B\log A\\ \log X< X(对所有的 X>0都成立) \]

  • 级数

\[ \sum_{i=0}^{N}{A^i}\ =\ \frac{A^{N+1}-1}{A-1} \]

  • 模运算

如果\(N\)整除\(A-B\)，那么我们就说\(A\)与\(B\)模\(N\)同余，记作\(A\equiv B(mod\ N)\)。满足运算性质：若\(A\equiv B(mod\ N)\)，则\(A+C\equiv B+C(mod\ N)\)，以及\(AD\equiv BD(mod\ N)\)。

（定理1.3） \[ 如果\ N\geq1，则\sum_{i=1}^{N}{i^2}=\frac{N(N+1)(N+2)}{6} \]

数学基础

全书将使用下面四个定义：

定义：如果存在正常数 \(c\) 和 \(n_0\) 使得当 \(N\ge n_0\) 时 \(T(N) \le cf(N)\)，则记作 \(T(N)=O(f(N))\)。

定义：如果存在正常数 \(c\) 和 \(n_0\) 使得当 \(N\ge n_0\) 时 \(T(N) \ge cg(N)\)，则记作 \(T(N)=\Omega(f(N))\)。

定义：\(T(N)=\Theta(h(N))\)，当且仅当 \(T(N)=O(h(N))\) 且 \(T(N)=\Omega(h(N))\)。

定义：如果 \(T(N)=O(p(N))\) 且 \(T(N)\ne\Theta(p(N))\)，则 \(T(N)=o(p(N))\)。

我们需要掌握的重要结论有：

法则1：

如果 \(T_1(N)=O(f(N))\) 且 \(T_2(N)=O(g(N))\)，那么

(a)\(T_1(N)+T_2(N)=max( O(f(N)), O(g(N)))\)，

(b)\(T_1(N)*T_2(N)=O(f(N)*g(N))\)。

法则2：

如果 \(T(N)\) 是一个 k 次多项式，则 \(T(N) = \Theta(N^k)\)。

法则3

对于任意常数 k，\(\log^kN=O(N)\)。它告诉我们对数增长十分缓慢。

模型

为了在正式的框架中分析算法，我们需要一个计算模型。我们的模型是一台标准的计算机，可以执行一些标准的指令，比如加法、乘法、比较和赋值。但不同于真实情况的是，模型机做任何一件简单的工作都恰好消耗一个时间单元。为了合理期间，我们假设模型有固定范围的整数并且不存在诸如矩阵求逆或排序等运算。我们还假设模型机有无限的内存。

要分析的问题

要分析的最重要的资源一般来说就是运行时间。我们主要考虑 \(T_{worst}\)，因为很难界定什么是“平均”，导致分析 \(T_{avg}\) 十分困难。

方法3：分而治之

这里采用“分治(divide-and-conquer)”策略。其想法是把问题分成两个大致相等的子问题，然后递归地对它们求解，这是“分”的步骤。“治”阶段则会将两个子问题的解合到一起并可能再做些少量的附加工作，最后得到整个问题的解。

常见的错误

memory access violation 和 segmentation violation

这种信息表示指针变量包含了伪地址。一个通常的原因是初始化变量失败。不论何时当需要确定指向的时候，就需要保证该指针不是 NULL。

何时使用 malloc

声明指向一个结构的指针并不创建该结构，只是给出足够的空间容纳结构可能会使用的地址。创建未被声明过的记录的唯一方法是使用 malloc 库函数。malloc 可以使系统创建一个新的结构并返回指向该结构的指针。另一方面，要是使用一个指针沿着表前进，就不需要创建新的结构了，也就不需要 malloc 命令。当有些空间不需要时，可以用 free 命令收回，free(P) 的结果是：P 指向的地址没变，但其中的数据已经没有定义了。

如果从未删除过一个链表，那么调用 malloc 的次数就应该是链表的长度，有 header 的话再加 1。

基数排序

基数排序有时也叫卡式排序。

如果我们有 N 个整数，范围从 1 到 M，我们可以利用这个信息得到一种快速的排序，叫做桶式排序。我们留置一个数组，称之为 Count，大小为 M，并初始化为 0。于是，Count 有 M 个单元（或桶），开始时他们都是空的。当 \(A_i\) 被读入后 Count[\(A_i\)]增 1。在所有输入被读进后，扫描数组 Count，打印好排好序的表。该算法的时间花费是 \(O(M+N)\)。如果 \(M=\Theta(N)\)，那么桶排序的时间就是 \(O(N)\)。

基数排序是桶排序的一种推广。

基数排序是桶排序的扩展。它的基本思想是：将整数按位切割成不同的数字，然后按照每位分别比较。具体做法是：将所有待比较的数值统一为同样的数位长度，较短的数前面补零。然后，从最低位开始排序。对于整数来讲，最大的数是几位，排序就要进行几次。

栈（stack）是限制插入和删除只能在一个位置上的表，该位置是表的末端，叫做栈的项（top）。对栈的基本操作有 Push（进栈）和 Pop （出栈），前者相当于插入，后者则是删除。最后插入的元素可以通过使用 Top 程序在执行 Pop 之前进行考察。对空栈进行 Pop 或 Top 一般被认为是栈 ADT 的错误。另一方面，当运行 Push 时空间用进是一个实现错误，不是 ADT 的错误。

栈的数组实现

用一个数组实现栈是简单的。每一个栈都有一个 TopOfStack，对于空栈来说它是 -1。为了将某个元素压到该栈中。我们将 TipOfStack 加 1，然后置 Stuck[TopOfStack] = X，其中 Stack 就是代表具体栈的数组。为了弹出栈元素，我们置返回值为Stuck[TopOfStack] 然后对 TopOfStack 减 1。很多时候，Stack 和 TopOfStack 是代表一个栈的结构的一部分。使用全局变量和固定名字来表示数据结构几乎总是有害的，因为一个程序会用很多个栈。

值得注意的是，这些操作不仅可以以常数时间运行，而且这个常数是很小的。在某些机器上，若在带有自增或自减寻址功能的寄存器上操作，整数的 pop 和 push 都可以写成一条机器指令，所以在以后栈很可能就是最基本的数据结构之一。
